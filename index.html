CASE/06
Secure Credit Card Tokenization & Encryption Service
PCI-DSS-aligned PAN tokenization API — raw card numbers never persisted, only encrypted tokens and masked digits.
Crypto
+
  • Built a Flask REST API that encrypts credit card numbers (PANs) using Fernet/AES-256 symmetric encryption, generating a random non-guessable UUID token for each card — mirroring how real payment gateways tokenize PANs.
  • Enforced PAN masking (only last 4 digits ever displayed) and ensured raw card data is never written to disk or logs, existing only in memory during the encrypt/tokenize step.
  • Gated decryption behind token-based access only, with secrets and environment files excluded from version control — security-aware design aligned with PCI-DSS handling principles.
